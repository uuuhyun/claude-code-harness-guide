# 기여 가이드

이 저장소는 Claude Code 하네스·플러그인을 **일관된 템플릿**으로 문서화합니다. 새 항목을 추가하거나 기존 문서를 고칠 때 아래를 따라 주세요.

## 새 하네스 추가하기

1. [`TEMPLATE.md`](TEMPLATE.md)를 복사해 `harnesses/<하네스이름>.md`로 만든다.
   ```bash
   cp TEMPLATE.md harnesses/<하네스이름>.md
   ```
2. 각 섹션을 채우고, 안내용 HTML 주석(`<!-- ... -->`)은 지운다.
3. 상단 메타 표의 **카테고리**와 **검증 상태**를 반드시 정확히 표기한다.
4. [`README.md`](README.md)의 **카탈로그 표**에 한 줄을 추가한다.
5. (여력 되면) [`README.en.md`](README.en.md) 카탈로그에도 한 줄 추가.

## 지켜야 할 원칙

- **카테고리(층위)를 먼저 분류한다.** A(하네스 팩) / B(방법론) / C(규율) / D(네이티브). 애매하면 이슈로 논의.
- **검증 상태를 솔직하게.** 직접 써봤으면 `✅ 써봄`, 문서·코드만 확인했으면 `🔍 조사만`.
- **설치·발동은 붙여넣기 가능하게.** 명령어는 코드블록으로, 프롬프트 예시는 실제 입력 그대로.
- **주의·트레이드오프를 생략하지 않는다.** 토큰 비용, 다른 하네스와의 충돌, 유지보수 부담 등 실사용 함정을 반드시 적는다. 이게 단순 링크 모음과 다른 점이다.
- **출처를 남긴다.** 공식 저장소·문서 링크 필수.

## 문서 상태 표기

| 표기 | 의미 |
|---|---|
| ✅ 써봄 | 작성자가 직접 설치·사용해 검증함 |
| 🔍 조사만 | 공식 문서/코드/글을 조사해 정리함 (미검증) |
